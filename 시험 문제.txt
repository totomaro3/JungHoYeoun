※ 시나리오 : 게시판 시스템을 개발해야 합니다

<요구사항 및 주요기능>

1. 사용자 관리:

- 회원 가입: 사용자의 이름, 이메일, 비밀번호를 입력받아 회원 가입을 처리합니다.

- 로그인: 입력받은 이메일과 비밀번호를 확인하여 사용자를 인증합니다.

- 프로필 편집: 인증된 사용자는 자신의 프로필을 편집할 수 있습니다.

2. 게시물 관리:

- 게시물 작성: 사용자는 제목과 내용을 입력하여 게시물을 작성할 수 있습니다.

- 게시물 목록 조회: 모든 게시물을 조회합니다.

- 게시물 상세 조회: 특정 게시물의 상세 내용을 조회합니다.

- 댓글 작성: 사용자는 게시물에 댓글을 작성할 수 있습니다.

3. 추가 요구사항:

3-1. 사용자 개념:

-사용자 정보: 이름, 이메일, 비밀번호 등의 정보를 저장합니다.

-회원 가입 시 이메일 중복 확인을 구현합니다.

-로그인 후에는 인증된 사용자만 프로필 편집을 할 수 있습니다.

3-2. 게시물 개념:

-게시물 정보: 제목, 내용, 작성자, 작성 시간 등의 정보를 저장합니다.

-게시물 작성 시 작성자 정보와 작성 시간을 저장합니다.

-게시물 목록은 페이징 처리를 구현하여 한 번에 일정 개수의 게시물만 표시됩니다.

3-3. 댓글 개념:

-댓글 정보: 내용, 작성자, 작성 시간 등의 정보를 저장합니다.

-댓글은 게시물과 연결되어야 하며, 작성자와 작성 시간을 저장합니다.

<테스트케이스 조건>

1.사용자 관리 시스템 테스트 케이스 조건:

1-1. 회원 가입:

-유효한 사용자 정보로 회원 가입을 진행하는 경우

-중복된 이메일로 회원 가입을 시도하는 경우

-필수 입력 필드인 이름, 이메일, 비밀번호를 누락하고 회원 가입을 시도하는 경우

1-2. 로그인:

-유효한 이메일과 비밀번호로 로그인을 성공하는 경우

-잘못된 이메일 또는 비밀번호로 로그인을 시도하는 경우

1-3. 프로필 편집:

-인증된 사용자가 자신의 프로필 정보를 정상적으로 수정하는 경우

-비인증된 사용자가 프로필 편집을 시도하는 경우

2. 게시판 시스템 테스트 케이스 조건:

2-1게시물 작성:

-제목과 내용을 입력하여 게시물을 작성하는 경우

-제목이나 내용이 비어있어 게시물 작성에 실패하는 경우

2-2.게시물 목록 조회:

-모든 게시물을 정상적으로 조회하는 경우

-게시물이 없는 경우에도 목록을 정상적으로 처리하는지 확인하는 경우

3-3.게시물 상세 조회:

-특정 게시물의 상세 내용을 정상적으로 조회하는 경우

-존재하지 않는 게시물의 상세 내용을 조회하는 경우

3-4.댓글 작성:

-유효한 내용으로 게시물에 댓글을 작성하는 경우

-댓글 내용이 비어있어 작성에 실패하는 경우
